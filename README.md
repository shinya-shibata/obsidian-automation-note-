# Obsidian × Gemini 自己組織化ナレッジシステム

このシステムは、ローカルの **Obsidian Vault (保管庫)** に蓄積されたメモや疑問を Google Gemini API (無料枠) と連携させ、自律的・自己組織的に知識を整理、課題をプール、新たな学術・哲学的理論を自動創出するシステムです。

---

## 📂 ディレクトリ構成

お使いの Obsidian Vault のルートに、以下のフォルダ構造を作成してください（スクリプト起動時に存在しない場合は自動生成されます）。

```text
Vault/
├── 01_Questions/         # 質問用フォルダ（ここにMarkdownで疑問を書く）
├── 02_Answers/           # 回答用フォルダ（AIの回答が同じファイル名で保存される）
├── 03_Brainstorming/     # アイデア・日常メモ（パース対象）
├── 04_Templates/
│   └── prompt_template.md # 思考整理・回答用プロンプトテンプレート
└── 99_System/
    ├── unsolved_questions.md # 集約された「未解決問題のデータベース」
    └── arxiv_summary.md      # セレンディピティ機能による学術論文の調査結果
```

## 🛠️ ローカル環境セットアップ手順 (3分)

### 1. Pythonの確認
Python 3.10 以上がインストールされていることを確認してください。
```bash
python --version
```

### 2. スクリプトファイルの配置
本レポジトリからダウンロード、またはコピーした以下のファイルを**同じディレクトリ**に配置します。
- `main.py`
- `config.py`
- `parser.py`
- `gemini_client.py`
- `arxiv_client.py`
- `requirements.txt`

### 3. 依存ライブラリのインストール
ターミナルを開き、配置したディレクトリに移動して以下を実行します。
```bash
pip install -r requirements.txt
```

### 4. APIキーとVaultパスの設定
同じディレクトリに `.env` ファイルを新規作成し、以下の内容を記述します。

```env
# Google AI Studioから取得したAPIキー
GEMINI_API_KEY="YOUR_GEMINI_API_KEY"

# あなたのObsidian Vault（保管庫）の絶対パス、または相対パス
# 例: /Users/username/Documents/MyObsidianVault
OBSIDIAN_VAULT_PATH="./Vault"
```

## 🚀 使い方と運用プロセス

ターミナルからスクリプトを起動します：
```bash
python main.py
```

起動すると対話型メニューが表示されます：

### 【フェーズ1】質問・回答の自動処理とクレンジング
1. `01_Questions/` フォルダ内に、任意の名前でMarkdownファイル（例: `量子力学と意識.md`）を作り、本文に聞きたいことや思考の整理テーマを記述します。
2. メニューから `1` を実行します。
3. `02_Answers/` フォルダ内に、同名で回答が自動生成されます。元の質問用ファイルは中身が空に「クレンジング」されます（枠組みだけ残るため、次の質問にすぐ使えます）。

### 【フェーズ2】未解決問題の自動プール
1. 日常のメモやアイデアを `03_Brainstorming/` フォルダにMarkdownで記述します。
2. メモの途中に、いつでも **`?-`** から始まる疑問を書き込んでおきます。
   *例:* `?- 臓器記憶の分子生物学的プロセスを実証するにはどうすればよいか？`
3. メニューから `2` を実行すると、Vault内の全てのメモから `?-` 行がスキャンされ、重複を排除して `99_System/unsolved_questions.md` （問いのデータベース）に自動集約されます。

### 【フェーズ3】キーワード・メモからの新理論自動生成
1. `03_Brainstorming/` のノートに「Front Matter(YAML)でキーワード」を定義するか、または「## 印象に残ったメモ」セクションに箇条書きを記述します。
2. メニューから `3` を実行すると、それらの語彙群が高次元で化学反応を起こし、新しい学問的理論、先行研究クエリ、および想定される批判/限界をセットにした壮大な学術調Markdownが `03_Brainstorming/新理論_YYYYMMDD_HHMMSS.md` として自動生成されます。

### 【フェーズ4】arXiv自動学術検索＆3行要約
1. メニューから `4` を実行すると、自動プールされた未解決問題やキーワードから自律的に検索クエリを構築します。
2. arXiv APIを直接叩いて最新の最先端論文アブストラクトを収集し、Geminiが日本語の3行要約と「貢献度スコアリング(A〜C)」を行い、`99_System/arxiv_summary.md` に追記保存します。

### 【フェーズ5】2つの異なる理論・概念の構造的統合 (新機能)
1. メニューから `5` を選択します。
2. 統合したい2つの理論や概念（例：「熱力学第二法則」と「マルクス主義経済学」）をそれぞれ入力します。
3. 適用する統合アプローチ（構造的同型性発見、視点・レンズ適用、弁証法、アノマリーブレイクスルー、自動最適化）を選択します。
4. Geminiが世界最高峰の科学哲学者・理論モデラーとして、Step 1〜5の厳密な思考プロセス（構造抽出、同型性写像対比表、前提の対立、メタフレームワーク統合、新理論の命名とコア命題・数式ロジックイメージ）を辿り、検証・批判（先行研究、限界点）までパッケージングした極めて堅牢な新理論を `03_Brainstorming/統合理論_YYYYMMDD_HHMMSS.md` として自動生成します。

## 💡 無料枠(Free Tier)における安全性
本スクリプトの `gemini_client.py` には、**自動待機制御(4秒ウェイト)** が組み込まれており、Gemini APIの無料枠制限である **15 RPM** の範囲内で完全に・安全に・エラーなしで動作するように設計されています。大量のファイルを一括処理する場合も、クォータオーバーによる中断は起きません。
